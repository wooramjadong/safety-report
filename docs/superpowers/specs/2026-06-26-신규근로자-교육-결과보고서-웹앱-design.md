# 신규근로자 교육 결과보고서 생성 웹앱 — 설계 문서

## 배경 및 목적

건설현장에서 신규근로자 교육 후 작성하는 결과보고서를 자동 생성하는 웹앱. 기존에는 엑셀 템플릿(정기안전교육, 채용시/작업내용변경시 교육, 특별안전교육)을 현장마다 수동으로 채워 작성했음. 교육 직후 현장에서 기본정보와 사진만 입력하면 결과보고서가 자동으로 만들어지고, 웹에서 즉시 인쇄·수정·다운로드까지 가능해야 함.

템플릿 3종(`정기안전교육 탬플릿.xlsx`, `채용시 작업내용변경시 교육 탬플릿.xlsx`, `특별안전교육 탬플릿.xlsx`)은 고정이며 변경하지 않음. 각 템플릿의 셀 구조와 `{{placeholder}}` 토큰을 그대로 유지한 채, 입력값을 채워 넣는 방식으로 동작한다.

## 범위 (v1)

- 로그인/계정 관리 없음 (추후 현장별 DB·계정 기능 필요시 별도로 확장)
- 서버/DB 없음 — 생성한 보고서는 저장하지 않고, 매번 새로 작성해서 즉시 다운로드
- 모바일/PC 동등 지원 (반응형)

## 보고서 종류 및 입력 스키마

### 공통 입력 필드 (3종 모두 해당)

| 필드 | 입력 방식 | 비고 |
|---|---|---|
| 협력사 | 텍스트 입력 | |
| 공종 | 텍스트 입력 | 특별안전교육에서는 1~39번 항목 자동매칭의 키워드 소스로도 사용 |
| 교육일시 | 날짜, **오늘 날짜 자동입력**, 수정 가능 | |
| 교육시간 (시작/종료) | 시작시간 입력 → **종료시간 = 시작 + 2시간 자동계산**, 수정 가능 | |
| 교육방법 | 선택형: "강의식" / "토론식" | |
| 교육장소 | 텍스트, **기본값 "안전교육장"**, 수정 가능 | |
| 교육인원 | 텍스트/숫자 입력 | |
| 교육강사 | 텍스트 입력 | |
| 교육사진 | 이미지 업로드 1장 | 템플릿에 사진 자리가 1곳뿐이므로 1장만 지원 |

담당자/소장 결재란은 입력 필드로 받지 않음 — 출력 후 수기 결재하므로 폼에서 제외.

### 1) 정기안전교육

공통 필드만 입력받음. 법정 교육내용은 템플릿에 고정 텍스트로 이미 박혀 있어 추가 입력 불필요.

### 2) 채용시 및 작업내용변경시 교육

공통 필드 + 상단 토글: "채용시 교육" ↔ "작업내용 변경시 교육" (템플릿의 ■/□ 체크박스 셀에 대응). 교육내용 텍스트는 두 경우 동일하게 고정.

### 3) 특별안전교육

공통 필드에 더해 **작업유형(1~39) 선택 로직**:

1. 사용자가 "공종" 필드에 텍스트를 입력하면, 39개 작업유형별로 미리 정의한 키워드 매칭표를 검사하여 키워드가 포함된 항목을 자동으로 매칭·선택한다.
2. 자동 매칭된 항목명을 화면에 표시하고, 드롭다운(1~39 전체 목록)을 통해 사용자가 직접 다른 항목으로 변경할 수 있다.
3. 선택된 항목의 교육항목/교육내용은 읽기전용으로 화면에 표시되며, 보고서 생성 시 템플릿의 `{{교육항목}}` / `{{교육내용}}` 자리에 그대로 채워진다.
4. 키워드 매칭표는 39개 항목 각각에 대해 `{id, title, keywords[], content}` 형태의 JSON으로 앱에 내장한다 (별표5 작업유형 시트 1~39 내용 기반으로 사전 추출).
5. 매칭은 "키워드 포함 여부"로 판단하며, 여러 항목이 동시에 매칭될 경우 목록상 먼저 오는 항목을 기본 선택한다 (사용자가 드롭다운으로 바로 교정 가능하므로 우선순위 로직은 단순하게 둔다).

## 화면 흐름

1. **홈**: 보고서 종류 3개 중 선택 (정기안전교육 / 채용시·작업내용변경시 교육 / 특별안전교육)
2. **입력 화면**: 선택한 보고서 종류에 맞는 폼 표시 (위 스키마 기준), 사진 업로드 포함
3. **미리보기/다운로드 화면**: 템플릿과 동일한 레이아웃으로 A4 미리보기 렌더링 → 인쇄 / Excel 다운로드 / PDF 다운로드 / 수정하기(입력화면으로 복귀) 버튼 제공

## 출력 레이아웃 (인쇄/PDF/화면 공통)

- A4 세로(210mm × 297mm), 1페이지 내 작성, **상하좌우 여백 40mm**로 본문을 가운데 배치.
- 화면 미리보기 / 인쇄(`window.print()`) / PDF 다운로드 모두 동일한 A4 페이지 레이아웃을 사용해 보이는 그대로 출력되도록 한다.
- 여백 40mm 적용 시 본문이 1페이지를 넘어가면, 1페이지에 들어갈 때까지 여백을 점진적으로 축소한다 (예: 40mm → 35mm → 30mm ... 최소값까지). 템플릿 자체의 셀 구조·서식은 변경하지 않고, 페이지에 배치하는 외곽 여백만 조정한다.

## 엑셀/PDF/인쇄 생성 로직

- 템플릿 3개 파일(`.xlsx`)을 앱 정적 자산으로 포함.
- **Excel 다운로드**: 브라우저에서 `exceljs`(브라우저 빌드)로 템플릿을 로드 → 템플릿 내 `{{placeholder}}` 셀을 찾아 입력값으로 치환 → 사진은 해당 셀 영역에 이미지로 삽입 → `.xlsx` Blob 생성 후 다운로드. 템플릿의 틀(셀 구조, 서식, 고정 텍스트)은 그대로 유지하고 placeholder만 치환한다.
- **화면 미리보기 / 인쇄 / PDF**: 같은 입력 데이터로 템플릿과 동일한 레이아웃의 HTML을 A4 페이지 컨테이너(여백 40mm, 자동 축소 로직 포함) 안에 렌더링.
  - 인쇄: `window.print()`로 브라우저 인쇄 다이얼로그 사용.
  - PDF 다운로드: `html2canvas` + `jsPDF`로 미리보기 영역을 캡처해 PDF 파일로 다운로드.
- 서버/DB 호출 없음 — 모든 처리는 클라이언트에서 완료.

## 기술 스택

- Vite + React 기반 정적 SPA
- `exceljs` (브라우저 빌드) — 엑셀 생성
- `html2canvas` + `jsPDF` — PDF 생성
- Vercel/Netlify 등 무료 정적 호스팅에 배포, URL로 접근 (서버/DB 불필요)
